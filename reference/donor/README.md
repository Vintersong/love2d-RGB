# Donor Reference Code

This folder keeps the donor prototype pieces that may still be useful as implementation references.

These files are not part of the runnable game and should not be required by `main.lua` or active `src/` modules. Some original donor `require(...)` paths are intentionally left unresolved because these are idea references, not standalone runtime modules.

- `shipDesign.lua`: procedural shape-tile ship rendering ideas
- `resolution.lua`: adaptive canvas and ultrawide scaling ideas
- `inputBindings.lua`: debug/pattern sandbox input ideas
- `attack.lua`: projectile pooling, scheduled attacks, and particle trail ideas
